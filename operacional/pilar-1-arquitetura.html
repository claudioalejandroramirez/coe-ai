A Arquitetura de Referência responde como construímos: stack, padrões de solução (RAG, agentes, gateway), classificação de dados e regras do que pode ser construído sem escalonar aprovação. Tudo vive em ADRs e blueprints versionados.

O que definimos e documentamos

Matriz de tecnologia, blueprints, segurança de dados e fronteiras de aprovação

Stack de IA
Matriz de decisão de tecnologia
Quando usar GPT-4o vs Claude vs modelo open-source. Quando RAG é suficiente vs fine-tuning. Quando agente autônomo vs fluxo determinístico. Tudo em ADR escrito e versionado.
Padrões de solução
Blueprints reutilizáveis
Padrão RAG corporativo (chunking, embedding, vector DB, permissões). Padrão de agente (orquestrador, estado, ferramentas). Padrão de prompt management. Gateway de LLM centralizado.
Segurança de dados
Classificação × modelo permitido
Dado público → API externa. Dado confidencial → modelo self-hosted ou DPA assinado. Dado com PII → anonimização obrigatória antes de qualquer LLM. Alinhado com LGPD.
Fronteiras de aprovação
O que pode ser construído
Lista viva: aprovado por padrão / requer revisão de arquitetura / proibido sem aprovação do CoE Estratégico. Critério baseado em impacto e autonomia do sistema.
