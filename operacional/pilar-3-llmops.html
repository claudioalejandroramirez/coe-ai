Pilar 3

Visão do pilar

Infraestrutura que sustenta produção — não só demos

📌 Resumo para todos

O gateway centraliza chamadas, políticas e custo. A observabilidade cobre latência por etapa, qualidade de retrieval e drift. O pipeline de dados sustenta RAG com reindexação segura. Custo é tratado como disciplina de engenharia, não surpresa na fatura.

1 Gateway centralizado de LLM
∞ PoCs que morreram sem isso
$ Custo como disciplina de eng.
⚠️ Alerta de maturidade
CoEs que pulam o Pilar 3 acumulam PoCs que não sobrevivem ao contato com produção. Este pilar é a ponte entre “funcionou na demo” e “aguenta segunda-feira de manhã”.
Operação

Operação de IA em escala

Gateway, observabilidade, dados e custo — os quatro blocos operacionais

Gateway de LLM
Ponto central de todas as chamadas
Roteamento inteligente entre providers. Fallback automático. Rate limiting por projeto. Logging centralizado. Cache semântico para reduzir custo e latência.
Observabilidade
Métricas que importam em IA
Latência por etapa (embedding, retrieval, geração). Custo por sessão, usuário e feature. Qualidade de retrieval (MRR, NDCG). Feedback implícito e explícito. Drift de comportamento.
Pipeline de dados
RAG é tão bom quanto seus dados
Ingestão de fontes corporativas. Chunking com preservação semântica. Metadata enrichment com permissões. Reindexação atômica quando documentos mudam.
Gestão de custos
Custo é responsabilidade de arquitetura
Orçamento por projeto com alertas. Seleção de modelo por complexidade da tarefa. Batch processing para o que não é tempo real (tipicamente mais barato). Inventário de versões de modelos.
