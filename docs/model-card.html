📌 Para que serve este documento

O model card é o passaporte técnico de um modelo em produção. Ele responde às perguntas que surgem em incidentes, auditorias e onboarding de novos membros — sem precisar abrir o código. Exigido pelo CoE como critério de aceite do ARB. Deve ser mantido atualizado a cada mudança de versão de modelo.

Seção 08

Histórico de versões

Registro de mudanças relevantes — modelo base, prompt, dados, critérios

Versão Data Descrição da mudança Responsável
Model cards devem ser revisados a cada mudança de modelo e no mínimo a cada 6 meses.

O model card preenchido deve ser arquivado em Confluence → CoE IA → Model Cards e o link deve ser registrado no Linear vinculado ao card do projeto.
Um model card é pré-requisito obrigatório para aprovação no ARB. Mantenha-o atualizado — um model card desatualizado é tratado como ausente nas auditorias.
