⚠️ Pré-requisito de deploy
Este checklist deve ser preenchido pelo responsável técnico do projeto e revisado pelo CoE Operacional. Todas as 6 perguntas precisam de resposta "Sim" ou justificativa documentada. Projetos com risco Alto ou Crítico exigem aprovação do CoE Estratégico.

Seção 02

As 6 perguntas obrigatórias

Todas precisam de "Sim" ou de justificativa documentada para "Não / N.A."

01
A solução tem fallback se o modelo alucinar ou falhar?
Sem fallback, uma falha do modelo pode travar o fluxo do usuário ou expor resposta incorreta como verdade.
02
Como o sistema detecta que uma resposta está errada?
Alucinações silenciosas são o maior risco de sistemas RAG e generativos. Detecção ativa previne dano acumulado.
03
Existe logging suficiente para debugar um incidente?
Sem logs de prompt, resposta, contexto e latência, incidentes em produção se tornam indebuguáveis — exigindo rollback cego.
04
Foi testado com adversarial inputs (prompt injection)?
Sistemas de IA são alvos naturais de manipulação via prompt. Testes adversariais são obrigatórios antes de exposição a usuários externos.
05
O custo projetado por mês foi calculado e aprovado?
Modelos de LLM em produção podem gerar custos inesperados em escala. Custo surpresa na fatura é sinal de governança fraca.
06
Existe plano de rollback documentado?
Todo sistema em produção pode degradar. Um plano de rollback claro reduz tempo de incidente de horas para minutos.

Seção 04

Classificação de risco

Impacto do erro × autonomia do sistema — define o nível de aprovação exigido

O sistema toma decisões sem revisão humana?
O que acontece no pior caso se o sistema errar?
Consulte a matriz no Pilar 2 · Governança para determinar o quadrante.
📋 Nível de aprovação exigido
Baixo / Médio: aprovação do CoE Operacional (Tech Lead + Arquiteto).
Alto: aprovação do CoE Operacional + CoE Tático (Product Manager).
Crítico: aprovação plena — CoE Operacional + Tático + Estratégico (C-Level / DPO).
